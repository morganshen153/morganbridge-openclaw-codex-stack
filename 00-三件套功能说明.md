# MorganBridge Stack 三件套功能说明

这套文件是用于把任务从 OpenClaw 稳定交给连接中的 Codex 会话的第三方集成三件套。

它由 `morganshen153` 维护，不代表 OpenAI 或 OpenClaw 官方立场。

## 1. MorganBridge Runtime

它是整个方案的本地运行时。

可以把它理解成：
- 接收来自 OpenClaw 插件的任务
- 把任务转交给连接中的 Codex 会话
- 把结果再带回来
- 尽量延续同一个任务的上下文

它更像“桥”和“转接器”，不是给人直接聊天用的界面程序。

## 2. MorganBridge OpenClaw Plugin

它是 OpenClaw 一侧使用的插件。

可以把它理解成：
- 告诉 OpenClaw 怎么把任务交给 MorganBridge
- 在需要的时候触发桥接
- 把拿回来的结果送回 OpenClaw 当前对话

它负责的是“发起”和“接回”。

## 3. MorganBridge Receiver Skill

它是 Codex 一侧配合使用的 skill。

可以把它理解成：
- 告诉 Codex 这是 MorganBridge 转过来的任务
- 让回复保持短、准、连续
- 在多轮沟通里尽量沿着同一件事继续，而不是跑偏

它负责的是“接手后怎么继续”。

## 这三者合在一起能做什么

合起来之后，这三件东西分别对应：
- `morganbridge-runtime`：真正的本地桥接运行时
- `morganbridge-openclaw-plugin`：OpenClaw 端的发送入口
- `morganbridge-receiver-skill`：Codex 端的接收规则

所以这不是三个互相独立的小文件，而是一套配合使用的连接方案。

## 它现在已经适合做什么

当前这套更适合：
- 把明确任务从 OpenClaw 交给 Codex
- 让 Codex 回传执行结果
- 在同一个任务上继续多轮沟通

## 它故意不承担什么

当前这套不强调：
- 漂亮界面
- 复杂控制台
- 可视化任务编排
- 大规模分析和监控

它现在最主要的价值是：

`让 OpenClaw 和 Codex 之间真正形成稳定、可重复的任务交接。`
